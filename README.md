# (S)ingle responsibility Principe

Uma classe deve ter apenas um motivo para mudar

Beneficios:

- faciliade de manuntenção e eveolução do código
- Códigom limpor e claro
- Facilidade para desenvolvimento de testes
- Reduz acoplamentos
